Enterprise Secure Document Platform

## 1. Individual User

**Description**
An individual user accessing and managing their own secure documents.

**Primary Goals**
- Upload and access documents securely
- Use offline access on a single device
- Maintain privacy and security

**Permissions**
- Upload documents
- Download documents
- View own audit activity
- Use offline access (if licensed)

**Restrictions**
- Cannot manage other users
- Cannot view system-wide audit logs
- Cannot manage licenses

## 2. Enterprise User

**Description**
A user belonging to an organization with controlled access.

**Primary Goals**
- Securely access assigned documents
- Work online and offline
- Comply with organization security policies

**Permissions**
- Upload and download allowed documents
- View own audit activity
- Use offline access (if enabled by admin)

**Restrictions**
- Cannot manage users or licenses
- Cannot access organization-wide settings

## 3. Enterprise Admin

**Description**
Administrator responsible for managing an organization.

**Primary Goals**
- Control users and devices
- Monitor document access
- Enforce security policies

**Permissions**
- Add, remove, and manage users
- Assign roles and licenses
- View organization audit logs
- Revoke offline access

**Restrictions**
- Cannot access internal system configurations
- Cannot modify platform-wide security logic

## 4. System Admin (Internal)

**Description**
Internal platform administrator with full system visibility.

**Primary Goals**
- Ensure platform stability and security
- Monitor compliance and incidents

**Permissions**
- View all audit logs
- Monitor system health
- Trigger security alerts
- Manage compliance settings

**Restrictions**
- Does not access user document content
- Does not perform daily business operations
